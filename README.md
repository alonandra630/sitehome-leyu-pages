# sitehome-leyu-pages

本仓库用于归档和发布多个独立 HTML 页面。这些页面通常用于展示静态内容、工具页面、信息展示页或其他无需后端支持的网页。

## 项目简介

这是一个通用的静态页面归档仓库，不针对任何特定网站或域名。仓库中的 HTML 页面彼此独立，可以单独访问或部署。适合存放小型项目、临时页面、测试页面或长期维护的静态资源。

## 目录说明

```
├── index.html          # 仓库首页（可选）
├── pages/              # 存放主要 HTML 页面
│   ├── example-1.html
│   └── example-2.html
├── assets/             # 页面依赖的静态资源
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：存放所有独立 HTML 页面，可根据需要创建子目录。
- `assets/`：存放页面所需的 CSS、JavaScript、图片等资源文件。
- 根目录下的 `index.html` 为可选的入口页面，用于导航或说明。

## 页面归档说明

- 每个 HTML 页面均为独立可运行的静态文件，不依赖外部数据库或后端服务。
- 页面名称应尽量反映其用途或内容，便于识别。
- 如需更新或替换页面，请直接修改对应文件并提交。
- 不建议在此仓库中存放敏感信息或动态内容。

## 维护说明

- 本仓库由维护者不定期更新，欢迎提交 Pull Request 或 Issue 以协助改进。
- 新增页面时，请确保资源路径正确，避免跨目录引用问题。
- 如页面不再需要，建议先确认无其他依赖后再删除。
- 保持目录结构清晰，避免文件过多堆叠在根目录。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 使用任意浏览器直接打开 `pages/` 下的 HTML 文件即可预览。
3. 也可将仓库部署到 GitHub Pages 或任何静态托管服务，实现线上访问。
